# Submission: Weekly Update

**Due:** Every Tuesday, starting Thursday April 9 (first one)
**Submit via:** Team folder (no GitHub PR needed)

**Deck:** [Analytics Deep Dive](../classes/20260407_c20w11_tue_analytics-deep-dive/slides_c20w11_tue_analytics-deep-dive.pdf)

---

## Overview

Starting now, every team submits a short weekly update to your team folder. This is not optional. Several teams have gone dark on submissions, and I need visibility into what everyone is doing. This takes 10 minutes to write.

If you're building, you have something to say. If you have nothing to say, that's a signal we need to talk.

---

## What to Submit

Submit a markdown file to your team folder in the class repo:

```
teams/your-team-name/weekly_update_YYYY-MM-DD.md
```

Use the date of the Tuesday the update is due. For example: `weekly_update_2026-04-07.md`.

---

## What to Include

Your update should be about half a page. Cover four things:

1. **What we built last week** and why
2. **What we're building this week** and why
3. **Current user count**
4. **Biggest blocker** (if any)

---

## Schedule

- **First one due:** Thursday, April 9 (one-time exception — first update due Thursday, then Tuesdays from then on)
- **Ongoing:** Every Tuesday after that

---

## Notes

This is non-negotiable going forward. Half a page. Not a novel.

The "and why" is not optional. Anyone can list tasks. What I need to know is whether you're making deliberate decisions about what to build and what to skip.
