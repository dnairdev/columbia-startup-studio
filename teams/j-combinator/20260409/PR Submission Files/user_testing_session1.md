# User Testing — Round 2
**Session 1**

---

**Participant:** Ju-Ting Sun

---

## Key Observations

- Landing page only showed a sign-in option with no visible sign-up path — caused immediate confusion.
- During sign-up, borough preference was limited to a single selection; user wanted ranked or multi-select choices.
- Preference settings felt too rigid — user wanted to rank options rather than pick just one.
- User did not always have a specific time in mind and wanted to browse events freely without a time filter.
- No onboarding or usage guide was present — user was unaware of the left-swipe gesture to discard events.
- Accidentally triggered iPhone back-swipe navigation and had to restart the flow from the beginning.
- Selecting preferences during sign-up and then choosing mood again on the discover page felt redundant and tedious.
- Event descriptions were too brief — user wanted more detail before deciding on an event.
- User wanted to see all available event options, not just a curated subset.
