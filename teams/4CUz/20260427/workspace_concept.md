# Tripable — Workspace Concept (Draft)

**Goal:** Make group trip planning collaborative and low-stress by turning “the group chat” into a shared workspace with clear decisions.

## Mental Model

Each trip is a **workspace** with multiple **tabs**. Each tab behaves like a blank page/board where anyone can add components, react, and discuss asynchronously.

## Tabs (v1)

1. **Availability**
   - Everyone marks when they’re free (simple availability grid).
   - Output: highlighted overlap windows + “best date candidates”.

2. **Places & Activities**
   - Anyone can add destination / lodging / activity cards (with links).
   - Components: cards, tags, comment threads, votes, “shortlist” lane.

3. **Budget**
   - Propose budget ranges (per person + shared).
   - Track shared expenses + suggested splits.
   - Output: agreed budget + running total vs. budget.

4. **Itinerary**
   - Convert winning cards into a timeline (day-by-day / time blocks).
   - Output: a realistic plan (what/when/where) the group can execute.

## Collaboration Features (core)

- **Voting**: lightweight decisions without endless back-and-forth.
- **Comments**: keep context attached to the option being discussed.
- **Notifications**: nudge quiet members to participate (without spamming).
- **Roles** (optional): organizer / editor / viewer for groups that want structure.

## Success Metrics (draft)

- Faster time-to-decision on date + destination
- Higher participation rate (votes/comments per invited member)
- Fewer “organizer-only” trips (more shared contribution)
